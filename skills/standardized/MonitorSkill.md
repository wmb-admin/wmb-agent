# MonitorSkill

- 中文名称: 全链路日志监控
- 所属分组: （三）数据库&监控Skill
- 来源Prompt: 美业SaaS自动迭代智能体完整版Prompt
- 模板说明: 以下输入、输出、约束与交接项基于原始 Prompt 结构化整理，便于接入 Agent。

## 核心目标

收集前后端运行异常与请求失败情况，形成部署前阻断依据。

## 触发时机

- 前后端改动完成并可运行后
- 需要确认日志、控制台和网络层是否存在严重异常时

## 输入

- 当前迭代版本号
- 后端服务日志和接口调用记录
- 前端控制台报错、网络请求失败和资源加载情况
- ServBay 或其他运行监控信息

## 输出

- /doc/monitor-{version}.md
- 后端异常、前端报错、失败接口和时间线记录
- 严重异常阻断项与修复建议

## 硬性约束

- 必须记录异常信息、发生接口、时间和版本
- 前端需覆盖控制台报错、4xx/5xx、资源加载失败等问题
- 出现严重异常时必须阻断部署流程

## 执行清单

1. 抓取后端 ERROR、空指针、SQL、参数和权限异常
2. 监控前端控制台、网络请求和资源加载失败
3. 整理异常发生页面、操作链路和版本信息
4. 生成监控报告并标记是否允许部署

## 交接对象

- CodeRuleCheckSkill
- DevOpsSkill

## 原始 Prompt

后端日志监控：通过ServBay实时监控后端服务日志，捕获ERROR异常、空指针、SQL异常、参数异常、权限异常，记录异常信息、发生接口、时间、版本
前端监控：监控前端控制台报错、网络请求失败、4xx/5xx接口、资源加载失败，记录报错页面、操作链路、版本信息
生成监控报告，路径：/doc/monitor-{version}.md
出现严重异常，直接阻断部署流程，修复完成后方可继续
