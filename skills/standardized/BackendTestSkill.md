# BackendTestSkill

- 中文名称: 后端一体化测试
- 所属分组: （一）后端相关Skill
- 来源Prompt: 美业SaaS自动迭代智能体完整版Prompt
- 模板说明: 以下输入、输出、约束与交接项基于原始 Prompt 结构化整理，便于接入 Agent。

## 核心目标

对本次后端改动执行一体化测试、覆盖率统计并产出版本化报告。

## 触发时机

- 后端代码改动完成后
- 需要确认接口稳定性、异常场景和覆盖率达标时

## 输入

- 本次迭代涉及的后端代码变更
- 当前迭代版本号
- 接口清单、权限要求和边界条件
- 覆盖率门槛与豁免范围

## 输出

- 测试用例清单和执行结论
- 覆盖率统计结果
- /target/site/test-report/backend/{version}/index.html
- 未通过项、未覆盖代码和优化建议

## 硬性约束

- 必须覆盖正常、非法参数、空参、权限不足、数据不存在、边界异常、重复提交场景
- 最低行覆盖率不低于 80%
- Entity、DTO、VO、枚举、常量、简单 getter/setter、启动配置类可按规则豁免
- 测试失败或覆盖率不达标时必须阻断后续流程

## 执行清单

1. 为新增或修改接口设计全覆盖测试用例
2. 记录每条用例的场景、入参、期望、实际和结论
3. 执行 JaCoCo 覆盖率统计并整理未覆盖区域
4. 生成带版本号的 HTML 报告并输出阻断项

## 交接对象

- ApiDocSkill

## 原始 Prompt

针对本次新增/修改代码，设计全覆盖测试用例，必含：正常业务、参数非法/为空、权限不足、数据不存在、边界异常、重复提交场景
单条用例包含：测试场景、请求入参、期望结果、实际结果、测试结论
集成JaCoCo代码覆盖率测试，统计行、方法、类、分支覆盖率，最低行覆盖率≥80%
豁免覆盖率范围：Entity/VO/DTO等数据类、枚举/常量类、简单getter/setter、启动/配置类
生成带版本号的HTML测试报告，路径：/target/site/test-report/backend/{version}/index.html
报告包含：版本信息、测试概览、用例详情、覆盖率数据、未覆盖代码、优化建议
测试不通过/覆盖率不达标，终止后续流程，返回修复后重测
