# FrontendCodeWriteSkill

- 中文名称: 前端代码开发
- 所属分组: （二）前端相关Skill
- 来源Prompt: 美业SaaS自动迭代智能体完整版Prompt
- 模板说明: 以下输入、输出、约束与交接项基于原始 Prompt 结构化整理，便于接入 Agent。

## 核心目标

严格基于版本化接口文档实现前端页面、API 请求层和联调逻辑。

## 触发时机

- 接口文档已生成并确认版本号后
- 需要开发或改造前端页面和接口对接逻辑时

## 输入

- 当前迭代版本号
- /doc/api-doc-{version}.html
- FrontendCodeReadSkill 的上下文结果
- 页面需求、权限要求和错误码语义

## 输出

- 前端 API 请求层代码
- 页面功能实现与权限展示逻辑
- 底部前端版本号展示
- 供 FrontendUISkill 和 FrontendTestSkill 使用的页面实现结果

## 硬性约束

- 必须先读取对应版本接口文档再开发
- 不得自定义接口、字段或错误码含义
- 前后端版本必须完全一致
- 字段名、请求方式、权限要求必须与接口文档一致

## 执行清单

1. 核对接口版本、路径、方法、权限和校验规则
2. 生成对应版本的前端 API 调用层
3. 实现页面逻辑并完成字段映射
4. 在页面底部展示前端版本号并确认联调一致性

## 交接对象

- FrontendUISkill
- FrontendTestSkill

## 原始 Prompt

标准接口文档读取&使用流程：
① 获取当前迭代版本号，读取对应版本接口文档/doc/api-doc-{version}.html
② 核对接口版本、请求路径、请求方式、权限要求
③ 解析入参校验规则、出参数据结构、错误码含义
④ 按规范生成对应版本前端API请求层代码
⑤ 严格依据接口文档开发，绝不自定义接口/字段
按芋道风格实现页面功能，保证接口对接字段完全匹配
页面底部展示当前前端版本号，前后端版本保持一致
