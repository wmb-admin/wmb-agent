# DevOpsSkill

- 中文名称: 版本化自动化部署
- 所属分组: （四）代码规范&部署Skill
- 来源Prompt: 美业SaaS自动迭代智能体完整版Prompt
- 模板说明: 以下输入、输出、约束与交接项基于原始 Prompt 结构化整理，便于接入 Agent。

## 核心目标

执行版本化打包、部署、服务控制与回滚，输出最终交付结果。

## 触发时机

- 所有前置环节均通过后
- 需要执行版本发布、重启或回滚时

## 输入

- 当前迭代版本号
- 代码规范检查通过结论
- 后端打包与前端构建产物
- servbay-cli 或等价部署控制命令

## 输出

- 带版本号的 Git 提交信息或发布记录
- 部署、重启、状态检查和回滚结果
- 最终版本信息、文档路径和部署反馈汇总

## 硬性约束

- 所有前置测试、监控、规范检查通过前不得部署
- 部署动作必须与当前版本号绑定
- 必须支持按版本回滚并反馈服务状态
- 若存在未修复慢 SQL 或严重异常则禁止发布

## 执行清单

1. 执行版本化 Git 提交与构建打包
2. 停止、部署、启动或重启目标服务
3. 检查服务状态并验证是否需要回滚
4. 汇总版本信息、文档路径和部署结果给用户

## 交接对象

- IterationSummary

## 原始 Prompt

执行Git操作，提交信息标注当前迭代版本号
完成后端Maven打包、前端项目构建
通过servbay-cli实现服务停止、部署、启动、重启、状态查看
支持按版本一键回滚，保障服务稳定性
部署完成后，汇总全量版本信息、文档路径、部署结果反馈给用户
