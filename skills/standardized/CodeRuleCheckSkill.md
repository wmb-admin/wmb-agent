# CodeRuleCheckSkill

- 中文名称: 代码规范检查
- 所属分组: （四）代码规范&部署Skill
- 来源Prompt: 美业SaaS自动迭代智能体完整版Prompt
- 模板说明: 以下输入、输出、约束与交接项基于原始 Prompt 结构化整理，便于接入 Agent。

## 核心目标

检查代码健壮性、复用性和工程规范，排除上线前的实现风险。

## 触发时机

- 功能开发、自测和监控完成后
- 准备进入打包和部署前

## 输入

- 本次前后端改动代码
- 团队代码规范和复用要求
- 日志、异常、组件封装和公共方法抽取情况

## 输出

- 代码规范检查结果
- 待修复问题清单和建议
- 是否允许进入 DevOpsSkill 的结论

## 硬性约束

- 只检查健壮性和规范性，不替代业务验收
- 后端要关注注解、工具类、异常和日志规范
- 前端要关注组件封装、命名规范和重复代码
- 检查不通过时必须先修复再重检

## 执行清单

1. 审查后端注解、异常处理、日志和通用工具使用
2. 审查前端组件复用、命名、公共方法抽取和冗余代码
3. 整理规范问题与修复建议
4. 确认阻断项关闭后再允许进入部署环节

## 交接对象

- DevOpsSkill

## 原始 Prompt

仅检查代码健壮性与规范性，不校验业务逻辑
后端检查：注解规范使用、三方工具类合理使用、异常/日志处理规范、无重复造轮子
前端检查：可复用组件必须封装、无冗余重复代码、命名规范、通用方法抽离
检查不通过，返回修复后重检
